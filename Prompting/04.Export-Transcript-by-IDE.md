## 1. 정상 변환 (지원 단위: meter, feet, yard)

| 케이스 | 명령 | 기대 |
|--------|------|------|
| meter 기준 (Golden Master) | `python -m unit_converter "meter:2.5"` | 3줄, feet `8.2021`, yard `2.7340` |
| 1 meter | `python -m unit_converter "meter:1"` | `1.0000` meter / `3.2808` feet / `1.0936` yard |
| feet → 전 단위 | `python -m unit_converter "feet:10"` | 10 feet 기준 3줄 출력 |
| yard → 전 단위 | `python -m unit_converter "yard:1"` | 1 yard 기준 3줄 출력 |
| 소수 | `python -m unit_converter "meter:0.5"` | 4자리 반올림 출력 |
| 큰 값 | `python -m unit_converter "feet:1000"` | 동작 확인용 |

```text

2.5 meter = 2.7340 yard
```

## 2. 형식 오류 (E-FMT)

메시지: `Invalid format. Use unit:value (ex: meter:2.5)`

```bash
python -m unit_converter ""
python -m unit_converter "meter"
python -m unit_converter "abc"
python -m unit_converter "meter:"
python -m unit_converter ":2.5"
```

---

## 3. 숫자 오류 (E-NUM) — CLI는 parser 경유

메시지: `Invalid number: {값문자열}`

```bash
python -m unit_converter "meter:abc"
python -m unit_converter "meter:2.5.3"
python -m unit_converter "meter:1e2"    # float() 가능하면 통과
```

---

## 4. 음수 (E-NEG)

메시지: `Negative value not allowed`

```bash
python -m unit_converter "meter:-1"
python -m unit_converter "feet:-0.1"
```

## 5. 미지 단위 (E-UNIT)

메시지: `Unknown unit: {단위}`

```bash
python -m unit_converter "cubit:1"
python -m unit_converter "inch:12"
python -m unit_converter "kilometer:1"
```

## UI 구성 (요청하신 형태)

| 위젯 | 역할 |
|------|------|
| `QComboBox` | 단위 선택 (meter / feet / yard) |
| `QLineEdit` 또는 `QDoubleSpinBox` | 숫자 입력 |
| `QPushButton` ("Converter") | 변환 실행 |
| `QTextEdit` (read-only) | 변환 결과 여러 줄 표시 |

오류는 같은 textarea에 `Invalid format...`, `Negative value not allowed` 등 **기존 E-* 메시지**를 그대로 보여주면 CLI와 계약이 일치합니다.

---

## 권장 레이어 (CLI와 동일 패턴)

```text
[PyQt UI]  ← 새 모듈 (예: unit_converter/ui/main_window.py)
    │  단위·값 수집, 버튼 클릭
    ↓
parse(f"{unit}:{value}")          # input_parser
convert_all(unit, value)          # converter
format_conversion_output(...)     # output_formatter → QTextEdit.setPlainText
```

- **PyQt는 화면·이벤트만** — 변환·검증은 기존 모듈 재사용
- `cli.py`와 **같은 orchestration**, 출력만 `print` 대신 `QTextEdit`

---

## PyQt 패키지 선택

| 패키지 | 비고 |
|--------|------|
| **PyQt6** | 현재 권장 (Qt6) |
| PyQt5 | 여전히 많이 사용 |
| **PySide6** | Qt 공식 라이선스(LGPL), API 거의 동일 |

실습/배포에서는 **PySide6**도 흔히 씁니다. 둘 다 “PyQt 스타일” GUI로 같은 방식으로 만들 수 있습니다.

`requirements.txt`에는 현재 `pytest`만 있으므로, GUI 추가 시 `PyQt6` 또는 `PySide6`를 **별도 의존성**으로 넣어야 합니다.

## 웹 vs PyQt (이 프로젝트 맥락)

| | PyQt | Flask/FastAPI 웹 |
|--|------|------------------|
| 배포 | 데스크톱 앱 (Python + Qt 필요) | 브라우저만 |
| lecture PRD | CLI 가정 — 둘 다 **확장** | 동일 |
| 도메인 재사용 | ✅ 동일 | ✅ 동일 |
| 테스트 | UI는 수동/E2E; 도메인은 기존 pytest | API route 테스트 가능 |

## 주의할 점

1. **PRD P0 범위 밖** — 문서/traceability에 `U-GUI-*` 등을 추가하는 것이 C2C에 좋습니다.
2. **음수·빈 값** — `QDoubleSpinBox(min=0)`로 UI에서 막거나, parser에 맡기기 (후자가 CLI와 동일).
3. **단위 문자열** — combobox의 `meter`/`feet`/`yard`를 `parse`에 넘기기 전 `f"{unit}:{value}"` 조합.
4. **pytest 19건** — GUI 없이도 도메인·CLI는 그대로 PASS; GUI는 선택적 Boundary TC.

## 정리

- **PyQt로 만들 수 있고**, 선택형 단위 + 숫자 + Converter 버튼 + 아래 텍스트 창 UI는 Qt 위젯으로 구현하기 쉬운 패턴입니다.
- 핵심은 **PyQt에 변환 로직을 넣지 않고** `input_parser` / `converter` / `output_formatter`를 호출하는 것입니다.

구현까지 원하시면 **Agent 모드**로 전환해 주세요. PyQt6 vs PySide6 선호도도 같이 알려주시면 됩니다.

## Dual-Track 관점 (lecture가 PyQt를 말할 때 흔한 이유)

| Track | 현재 프로젝트 | PyQt 추가 시 |
|-------|---------------|--------------|
| **B (Logic)** | `test_converter.py` — parser/converter/registry | **그대로 재사용** |
| **A (Boundary/UI)** | `test_cli.py` — subprocess CLI | **새 Track A** — `test_gui.py` 또는 UI TC |

PyQt는 **새 Boundary(UI)** 이고, 도메인은 `input_parser` / `converter` / `output_formatter`를 **호출만** 하면 됩니다.  
→ **refactoring이 아니라「Boundary 확장」 또는 ⑥·⑦ 단계**입니다.
